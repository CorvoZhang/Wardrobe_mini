# 香奈儿风格UI组件库

## 1. 组件库概述

本组件库基于香奈儿品牌设计风格，提供了一系列简约、优雅、奢华的UI组件，适用于衣橱管理系统。组件库遵循香奈儿的经典黑白金配色方案，注重细节处理和用户体验，确保在不同设备上都有良好的表现。

## 2. 组件分类

### 2.1 基础组件
- 按钮（Button）
- 输入框（Input）
- 选择器（Select）
- 文本域（Textarea）
- 标签（Label）

### 2.2 布局组件

### 2.3 数据展示组件
- 统计卡片（Stat Card）
- 列表（List）
- 表格（Table）
- 图片（Image）

### 2.4 交互组件
- 模态框（Modal）
- 抽屉（Drawer）
- 通知（Notification）
- 加载状态（Loading）

## 3. 组件设计规范

### 3.1 按钮（Button）

#### 3.1.1 主按钮（Primary Button）
- **样式**：黑色背景，白色文本，圆角4px
- **尺寸**：
  - 大按钮：padding 12px 24px，字体16px
  - 中按钮：padding 8px 16px，字体14px
  - 小按钮：padding 6px 12px，字体12px
- **效果**：
  - 悬停：背景色变为深灰色，添加轻微阴影
  - 点击：背景色变为黑色，轻微下沉效果
- **使用场景**：主要操作，如提交表单、确认操作等

#### 3.1.2 次要按钮（Secondary Button）
- **样式**：白色背景，黑色边框，黑色文本，圆角4px
- **效果**：
  - 悬停：背景色变为浅灰色，边框颜色加深
  - 点击：背景色变为灰色，轻微下沉效果
- **使用场景**：次要操作，如取消、返回等

#### 3.1.3 金色按钮（Gold Button）
- **样式**：金色背景，白色文本，圆角4px
- **效果**：
  - 悬停：背景色变为深金色，添加轻微阴影
  - 点击：背景色变为更深的金色，轻微下沉效果
- **使用场景**：强调操作，如特殊功能、VIP功能等

### 3.2 卡片（Card）
- **样式**：白色背景，1px浅灰色边框，圆角8px
- **内边距**：24px
- **效果**：
  - 悬停：添加轻微阴影，轻微上浮效果
  - 过渡：平滑的过渡动画，持续时间0.3s
- **使用场景**：展示信息、统计数据、功能模块等

### 3.3 导航栏（Navigation）
- **样式**：白色背景，黑色文本，固定在顶部
- **高度**：80px
- **Logo**：左侧对齐，黑色文本，简洁字体
- **导航项**：水平排列，间距32px
- **活动状态**：底部添加1px金色下划线
- **效果**：
  - 滚动：背景色保持白色，添加轻微阴影
  - 悬停：文本颜色变为黑色，平滑过渡

### 3.4 统计卡片（Stat Card）
- **样式**：白色背景，1px浅灰色边框，圆角8px
- **内边距**：32px
- **布局**：
  - 顶部：描述文本（ uppercase, 12px ）
  - 中部：标题（ 22px, 衬线字体 ）
  - 右侧：图标（ 48px × 48px, 黑色背景 ）
  - 底部：数值（ 48px, 粗体 ）
- **效果**：
  - 悬停：添加轻微阴影，轻微上浮效果
  - 顶部显示1px金色线条

### 3.5 表单元素

#### 3.5.1 输入框（Input）
- **样式**：白色背景，1px浅灰色边框，圆角4px
- **内边距**：12px 16px
- **字体**：16px，无衬线字体
- **效果**：
  - 聚焦：边框颜色变为金色，添加轻微金色光晕
  - 悬停：边框颜色变为深灰色

#### 3.5.2 标签（Label）
- **样式**：黑色文本，14px，半粗体
- **间距**：底部8px
- **文本转换**： uppercase，字母间距1px

### 3.6 图片组件
- **样式**：圆角8px，1px浅灰色边框（可选）
- **效果**：
  - 悬停：轻微放大效果（ 1.05倍 ）
  - 过渡：平滑的过渡动画，持续时间0.3s
- **加载状态**：灰色渐变背景，骨架屏效果

## 4. 组件使用示例

### 4.1 按钮组件
```jsx
// 主按钮
<button className="ant-btn-primary">提交</button>

// 次要按钮
<button className="ant-btn-default">取消</button>

// 金色按钮
<button className="ant-btn-accent">立即体验</button>
```

### 4.2 卡片组件
```jsx
<div className="card">
  <h3 className="card-title">卡片标题</h3>
  <p className="card-content">卡片内容</p>
</div>
```

    </div>
  </div>

</div>
```

  </div>

```

## 5. 响应式设计规范

### 5.1 断点设置
| 设备类型 | 断点宽度 | 布局调整 |
|----------|----------|----------|
| 移动设备 | < 768px | 单列布局，垂直导航，简化元素 |
| 平板设备 | 768px - 1024px | 双列布局，简化导航，调整间距 |
| 桌面设备 | > 1024px | 多列布局，完整导航，标准间距 |

### 5.2 响应式组件调整
- **导航栏**：在移动设备上转换为汉堡菜单
- **卡片布局**：从多列变为单列
- **字体大小**：根据屏幕尺寸调整字体大小
- **间距**：在小屏幕上减小间距，优化空间利用

## 6. 代码实现指南

### 6.1 CSS架构
- **CSS变量**：使用CSS变量管理颜色、字体、间距等
- **模块化**：采用模块化的CSS架构，如BEM或CSS Modules
- **响应式设计**：使用媒体查询实现响应式布局
- **性能优化**：避免过度使用CSS动画和复杂选择器

### 6.2 组件实现
- **组件化**：将UI拆分为可复用的组件
- **状态管理**：合理管理组件状态，确保性能
- **可访问性**：确保组件符合WCAG可访问性标准
- **测试**：编写组件测试，确保功能和样式正确

### 6.3 性能优化
- **懒加载**：图片和组件懒加载
- **代码分割**：将代码分割为多个 chunks
- **缓存策略**：合理的缓存策略，减少重复请求
- **优化渲染**：避免不必要的重渲染

## 8. 最佳实践

### 8.1 设计原则
- **简约至上**：简洁的线条，清晰的层次结构
- **对称平衡**：注重视觉平衡与比例协调
- **高品质感**：精致的细节处理，优雅的过渡效果
- **一致性**：统一的设计语言贯穿整个应用
- **用户体验优先**：在美学基础上确保良好的功能性和易用性

### 8.2 色彩使用
- 主要使用黑白金经典配色
- 金色仅用于强调元素，避免过度使用
- 辅助色用于补充信息和增强层次感
- 保持色彩使用的克制，确保整体视觉的纯净与优雅

### 8.3 排版使用
- 标题使用衬线字体，体现优雅奢华
- 正文使用无衬线字体，确保清晰可读
- 保持字体层级清晰，确保良好的阅读体验
- 合理使用字间距和行高，提升文本可读性

## 9. 常见问题与解决方案

### 9.1 组件样式冲突
- **解决方案**：使用CSS Modules或BEM命名规范，确保样式隔离

### 9.2 响应式设计问题
- **解决方案**：使用媒体查询和弹性布局，确保在不同设备上都有良好的表现

### 9.3 性能问题
- **解决方案**：优化组件渲染，使用懒加载和代码分割，减少不必要的重渲染

### 9.4 可访问性问题
- **解决方案**：确保组件符合WCAG可访问性标准，使用适当的ARIA属性
