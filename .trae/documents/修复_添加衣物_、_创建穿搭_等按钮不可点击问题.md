# 修复"添加衣物"、"创建穿搭"等按钮不可点击问题

## 问题分析
经过系统排查，发现按钮不可点击的主要原因如下：
1. **缺少axios集中配置**：没有设置请求拦截器自动携带认证token
2. **认证状态管理缺失**：页面组件没有检查用户认证状态
3. **按钮点击事件无实际功能**：部分按钮只有console.log，没有实际跳转或弹窗
4. **缺少API请求处理机制**：无法正确处理API响应和错误

## 修复方案

### 1. 创建axios配置文件
- **文件路径**：`client/src/utils/axiosConfig.js`
- **功能**：
  - 创建axios实例并配置baseURL
  - 添加请求拦截器，自动从localStorage获取并携带token
  - 添加响应拦截器，统一处理API响应和错误

### 2. 实现认证状态管理
- **修改MainLayout组件**：
  - 添加useEffect检查localStorage中的认证信息
  - 实现登出功能，清除localStorage中的token和用户信息
  - 添加认证状态上下文（Context）供所有页面组件使用

### 3. 修复按钮点击事件
- **Closet.jsx**：完善`handleAddClothing`函数，实现跳转到添加衣物页面
- **Outfits.jsx**：完善`handleAddOutfit`函数，实现跳转到创建穿搭页面
- **其他相关页面**：确保所有按钮都有实际功能

### 4. 添加API调用功能
- **修改各页面组件**：将模拟数据替换为实际API调用
- **实现错误处理**：添加适当的错误提示和加载状态

## 预期效果
- 所有按钮在用户登录后可正常点击
- 点击按钮后执行相应功能（跳转或弹窗）
- API请求自动携带认证信息
- 统一处理API响应和错误
- 未登录用户会被重定向到登录页面

## 修复步骤
1. 创建axios配置文件
2. 实现认证状态管理
3. 完善按钮点击事件
4. 添加API调用功能
5. 测试所有相关功能
